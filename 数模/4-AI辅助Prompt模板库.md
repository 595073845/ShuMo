# Prompt 模板库

> **目标**: 比赛中快速调用，不用现场组织语言  
> **准备**: 阶段三整理最终版，赛前确认可用

---

## 一、概念学习类（阶段一用）

### 解释概念
```
用大一能听懂的方式解释[模型名称]。
用类比+一个具体例子说明。
```

### 对比模型
```
对比[模型A]和[模型B]：
1. 适用条件
2. 数据需求
3. 计算复杂度
4. 优缺点
用表格形式呈现。
```

### 理解推导
```
用最直觉的方式解释[公式/算法]的核心思想，
不要追求严格推导，讲清楚"它在做什么"。
```

## 二、模型选择类（比赛核心）

### 模型推荐
```
我有一个问题：
- 目标：[描述要解决什么]
- 数据：[N]条，[M]个特征
- 输出：[连续值/离散类别/排序/方案]
- 约束：[已知约束条件]

推荐3个可用模型，从精度、复杂度、数据需求对比。
```

### 可行性判断
```
我选了[模型名称]来解决[问题描述]。
帮我判断：
1. 这个选择是否合理？
2. 有没有更适合的模型？
3. 使用时需要注意什么问题？
```

### 已有模型改进
```
我已经用了[模型名称]得到[结果]，
但[不满意的地方]。
有什么改进方向？
```

## 三、代码调试类（比赛高频）

### 调包代码生成
```
用Python实现[模型名称]。
数据格式：X是[形状]的数组，y是[形状]的数组。
要求：
1. 包含数据预处理
2. 模型训练+预测
3. 可视化结果
4. 计算评价指标
```

### Bug修复
```
这段代码报错[错误信息]。
帮我找到问题并修复。
代码：
```python
[粘贴代码]
```
```

### 出图美化
```
基于我的数据[数据描述]，
用matplotlib或plotly画一张适合放在学术论文中的图。
要求：简洁、清晰、有标题、有坐标轴标签。
```

## 四、论文写作类（阶段三核心）

### 思路梳理
```
帮我梳理[问题]的论文写作思路。
问题描述：[粘贴问题]
已用模型：[模型名称]
得到结果：[结果]
我需要写：问题分析、模型建立、结果分析三部分。
```

### 段落润色
```
帮我润色以下段落，要求：
1. 保持原意
2. 提升学术性和流畅度
3. 保留技术细节
4. 不超过原文长度

原文：
[粘贴自己的文字]
```

### 摘要修改建议
```
这是我要提交的论文摘要草稿。
请从以下几点给出修改建议：
1. 逻辑结构是否清晰？
2. 每个问题是否都提到了？
3. 有没有显得空泛的表述？
4. 改进方向

摘要：
[粘贴摘要]
```

### 英文摘要翻译
```
将以下中文摘要翻译为学术英文。
要求：准确、用词正式、符合学术规范。

[粘贴中文摘要]
```

## 五、LaTeX排版类

### 表格生成
```
生成一个LaTeX表格，3列5行：
| 列1名 | 列2名 | 列3名 |
| 数据1 | 数据2 | 数据3 |
...
```

### 公式输入
```
帮我写出这个公式的LaTeX代码：
[用文字描述公式]
```

## 六、比赛中专用Prompt

### 第一天选题后
```
题目：[粘贴题目关键信息]
我们已经确定选这道题。

请帮我：
1. 分析题目中的关键信息
2. 建议可能的建模方向
3. 需要注意的数据处理要点
```

### 第二天卡住时
```
我们在[问题描述]上卡住了。
目前尝试了[已尝试的方法]，但是[遇到的问题]。
有什么其他思路可以尝试？
```

### 第三天检查
```
作为数学建模竞赛阅卷老师，请帮我检查这段摘要：
[粘贴摘要]
存在什么问题？如何改进？
```

## 使用建议

### 模板不是死板的
根据具体情况修改，核心是保持"问题+上下文+格式要求"的结构。

### 建立自己的模板库
阶段二和三中，将好用的prompt加入这篇笔记。

### 不要只用一个AI
- 概念解释类 → 任何大模型都可以
- 中文润色 → 用Kimi/文心一言（中文效果好）
- 代码debug → ChatGPT/Claude
- 英文翻译 → DeepL
