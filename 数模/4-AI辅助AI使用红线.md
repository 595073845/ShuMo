# AI 使用红线

> **最重要的事**: CUMCM 竞赛允许使用 AI，但必须**明确标注**  
> 违反规则可能导致取消成绩

---

## ✅ 可以做

| 用途 | 说明 |
|------|------|
| 解释概念 | 让AI讲清你不懂的模型原理 |
| 对比模型 | 让AI从维度对比不同模型的优缺点 |
| 生成代码框架 | 让AI生成初始代码，你修改验证 |
| 代码debug | 贴报错信息让AI帮助定位问题 |
| 润色段落 | 你写完后让AI优化表达（不改变原意） |
| 校对英文摘要 | 用DeepL翻译后人工校对 |
| 头脑风暴 | 让AI给出几种模型选择的思路 |
| 搜索类似文献 | 让AI帮你找相关方法 |
| LaTeX/排版辅助 | 让AI生成表格、公式代码 |
| 检查摘要 | 让AI从阅卷角度提修改建议 |

---

## ❌ 不可以做

| 用途 | 风险 |
|------|------|
| AI直接生成论文正文 | **查重必过不去** |
| AI生成模型结果/结论 | 结果可能完全错误，必须基于实际运行数据 |
| 完全不验证AI写的公式 | 公式可能不对，比赛丢分是小事，导致错误结论更严重 |
| 照搬AI生成的表述 | 查重会检测到AI生成的内容 |
| 让AI生成数据 | 编造数据是学术不端 |
| 让AI做最终决策 | 选题、模型选择、结果判断必须人做 |

---

## 查重规避指南

### 为什么需要规避
- CUMCM 论文提交后会有查重
- AI生成的内容在查重中可能被标记（虽然组委会对AI查重还在探索阶段）
- 但 **不要冒这个风险**

### 如何安全使用AI写作

```
1. 自己写初稿（用自己的语言表达）
2. 让AI润色（保持原意，提升表达）
3. 对比前后版本，确认没有改变意思
4. 把AI改写的句子再读一遍，改成自己的话
```

### 特别危险的场景
- ❌ **摘要直接让AI写** → 摘要查重最严格，必须自己写
- ❌ **模型描述从AI复制** → 容易被识别
- ❌ **结果分析AI生成** → 数据和结论必须基于你的代码运行结果

## 竞赛标注要求

如果比赛明确要求声明AI使用情况：

```
本论文在以下环节使用了AI工具：
1. [工具名] 用于代码调试
2. [工具名] 用于英文校对
```

**具体看当年竞赛通知的要求**，有要求就如实标注。

---

## 团队AI使用公约

建议三人一起确认：

- [ ] 我们都理解AI的正确使用方式
- [ ] 我们都不会让AI直接写论文
- [ ] 我们都同意"AI的输出必须人工验证"
- [ ] 我们知道查重的风险并会规避
- [ ] 我们约定了一个AI使用中的"紧急叫停机制"
   （如果某人在让AI做明显越界的事，其他人有义务叫停）

---

## 最后的提醒

> **AI是你的副驾驶，不是你的飞行员。**  
> 比赛的荣誉属于你，论文的署名是你，  
> 所以核心工作必须由你完成。  
> AI节省下来的时间应该花在**思考**上，而不是花在**偷懒**上。
