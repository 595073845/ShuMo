# 竞赛72小时 AI 协作节奏 SOP

> **目标**: 人做关键决策，AI加速执行  
> **原则**: 人不该省的时间不省（读题、思考、判断结果合理性）  
>         该省的时间让AI省（调包、debug、格式化、翻译）

---

## Day 1：选题 + 定方向（人主导，AI辅助）

### 上午（8:00-12:00）— 读题
```
【人】
- 每人独立逐字读题
- 圈出关键信息：数据、目标、约束、问题数量
- 三人讨论：各题的理解、难度、数据可得性
- 选题表决

【AI】— 对应 [[Prompt模板库]] #六
- "帮我分析题目的关键信息"
- "建议可能的建模方向"
```

### 下午（14:00-18:00）— 定模型方向
```
【人】
- 建模手画模型框架图
- 确定：用哪些模型、模型之间关系、输入输出是什么
- 确定数据需要怎么处理

【AI】
- "我有[描述]的数据，推荐3个模型，对比优缺点"
- 生成数据预处理代码框架
```

### 晚上（20:00-22:00）
```
【人】
- 写作手开始写引言+问题重述框架
- 编程手开始写数据读取+清洗代码

【AI】
- 润色写作手写的段落
- 协助编程手debug
```

---

## Day 2：建模 + 求解（AI辅助主力）

### 全天 — 核心工作
```
【人】— 最重要的时间
- 建模手推导核心公式、确定模型细节
- 编程手实现模型代码
- 写作手写模型描述（一边写一边和建模手沟通）

【AI】
- 代码调包："用scipy.optimize实现线性规划"
- 代码debug：报错贴给AI
- 出图："画一个3D散点图，带颜色分类"
- 写作：生成LaTeX表格、格式化模型公式

【节奏点】每1-1.5小时站起来休息5分钟
```

### 注意事项
- ❌ 不要让AI生成整个模型（它可能生成错的）
- ❌ 不要完全信任AI写的公式（必须人工验证）
- ✅ 每用一次AI，验证一下结果是否合理

---

## Day 3：整理 + 论文冲刺（人主导）

### 上午（8:00-12:00）— 结果整理
```
【人】
- 判断所有结果是否合理
- 确定最终方案
- 设计呈现方式（什么用图、什么用表）

【AI】
- 敏感性分析代码
- 表格格式化
- 图的美化调整
```

### 下午（14:00-18:00）— 论文核心
```
【人】— 最关键的时间
- 写作手主笔写正文
- 建模手提供模型描述
- 三人集中精力写摘要！！！

【AI】
- 润色写作手已经写好的段落
- 校对英文摘要（DeepL翻译后人工校对）
- LaTeX编译报错修复
```

### 晚上（18:00-22:00）— 收尾
```
【人】
- 最后一遍通读全文
- 三人各自检查（找错别字、逻辑漏洞）
- 排版调整

【AI】
- "作为阅卷老师，检查这篇摘要有什么问题"
- 参考文献格式整理
- 最终转PDF前的检查
```

## AI使用节奏

| 时段 | 人机比例 | 说明 |
|------|---------|------|
| Day1 上午 | 9:1 | 读题必须人自己做 |
| Day1 下午 | 7:3 | AI提供模型思路参考 |
| Day2 全天 | 5:5 | AI在编码端大量介入 |
| Day3 上午 | 7:3 | 人判断结果，AI辅助呈现 |
| Day3 下午 | 6:4 | 人主笔写作，AI辅助润色 |
| Day3 晚上 | 8:2 | 人做最终检查 |

---

## 需要避免的情况

- ❌ **Day1 还没定方向** → 不要让人AI帮你选（模型可以，选题不行）
- ❌ **Day2 代码跑不通** → AI是debug利器，但如果AI改了三轮还不行，换方案
- ❌ **Day3 摘要还没开始写** → 这是最危险的情况，放下一切先把摘要写完
- ❌ **AI过度依赖综合症** → 验证验证再验证，AI给的结果必须人脑过一遍
